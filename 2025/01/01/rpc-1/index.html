RPC 总体概述

第一部分：RPC 总体概述

  1. RPC 的起源与发展背景
    • 分布式系统和服务化的演进历程
    • 从 SOA 到微服务的变化
    • RPC 在分布式架构中的地位
  2. 什么是 RPC（Remote Procedure Call）
    • RPC 的基本定义
    • RPC 与 HTTP RESTful、消息队列等常见通信方式的对比
    • RPC 的优缺点
  3. RPC 的核心概念和流程
    • Client 与 Server 的角色划分
    • Stub/Proxy 的作用
    • 请求/响应的数据封装过程
    • 消息的序列化、反序列化
    • 传输协议与通信机制
  4. RPC 在现代软件架构中的应用场景
    • 微服务化场景
    • 大规模分布式业务系统
    • 低延迟、高并发需求

第二部分：RPC 核心原理详解

  1. 整体通信模型
    • 同步阻塞通信模型
    • 异步非阻塞（NIO）模型
    • 双向流式通信（如 gRPC 的流式通信模式）
  2. 序列化与反序列化（序列化协议）
    • 常见序列化协议：Java 原生、JSON、Protobuf、Thrift、Avro、Hessian 等
    • 如何选择合适的序列化协议
    • 序列化的兼容性与扩展性
  3. 传输层实现与网络协议
    • TCP、HTTP/2、QUIC 等传输协议选型
    • 基于 Netty、gRPC 底层传输机制解析
    • 长连接与短连接的取舍
  4. 服务发现与注册中心
    • 静态配置 vs 动态发现
    • 服务注册与心跳检测
    • 常见注册中心（Zookeeper、Consul、Etcd、Nacos 等）
  5. 负载均衡与路由策略
    • 轮询、随机、一致性哈希、权重等常见算法
    • 服务端负载均衡 vs 客户端负载均衡
    • 流量调度策略与熔断限流机制
  6. 服务治理与监控
    • RPC 调用中的指标收集（延迟、吞吐量、错误率等）
    • 分布式追踪（Tracing）与日志协同
    • RPC 服务的熔断、限流与降级
  7. 安全性与权限控制
    • 传输安全（TLS/SSL、HTTP/2 中 TLS 的应用）
    • 客户端与服务端的认证与授权
    • 数据加密与合法性校验

第三部分：RPC 功能模块的设计与实现

  1. 客户端 Stub/Proxy 的实现
    • 动态代理 vs 静态代理
    • 代码生成（编译时生成、运行时生成）的优劣势
    • 如何屏蔽底层通信细节
  2. 服务端框架与请求分发
    • 服务端处理请求的线程模型（单线程、多线程、线程池）
    • 服务调度流程（进程内调度、应用容器调度）
    • 业务逻辑与 RPC 调用的解耦
  3. 多语言支持与跨语言调用
    • 不同语言中的协议一致性挑战
    • IDL（Interface Definition Language）的应用（如 Protobuf、Thrift）
    • 保持服务接口的兼容与版本管理
  4. 异常处理与重试机制
    • 网络波动与超时重试
    • 端到端异常的传递
    • 幂等性保障与容错设计
  5. 配置中心与动态配置
    • 配置推送与热加载
    • 客户端与服务端的参数调整
    • 统一管理与配置回滚

第四部分：主流 RPC 框架分析

  1. gRPC
    • 基于 HTTP/2 和 Protobuf 的核心特性
    • 双向流式调用和多种调用模式（Unary、Server streaming、Client streaming、Bidirectional）
    • 服务发现与负载均衡机制（官方 + 环境集成）
    • 应用案例：微服务之间的高速通信
  2. Dubbo
    • Dubbo 的历史与核心功能（高性能 RPC、服务发现、服务治理等）
    • 底层通信机制与扩展点
    • Dubbo 与 Spring/Spring Boot 的整合
    • Dubbo 3.x 对多语言、Triple 协议的支持
  3. Apache Thrift
    • Thrift IDL 与跨语言特性
    • 同步/异步服务模型
    • 对比 gRPC 在性能与协议层面上的差异
    • 典型应用场景
  4. Hessian
    • Hessian 协议特点（序列化的高效性、跨语言能力）
    • 适用场景与局限性
    • 与其他二进制序列化协议的对比
  5. 其他
    • Motan、Finagle、Tars、BRPC 等
    • 各自的主要应用场景与生态

第五部分：RPC 实战与最佳实践

  1. 项目实践案例：自研 RPC 框架如何落地
    • 架构设计与技术选型
    • 功能模块的分层与职责划分
    • 性能压测与优化策略
  2. 集成 CI/CD 与灰度发布
    • RPC 服务的自动化测试与持续集成
    • 分批灰度与蓝绿发布
    • 回滚策略与可观测性（Observability）
  3. 分布式追踪与全链路监控
    • 使用工具（Zipkin、Jaeger、SkyWalking 等）进行调用链路跟踪
    • 常见监控指标收集：QPS、延迟、错误率
    • 异常告警与故障定位
  4. 故障演练与高可用设计
    • 容错机制：熔断、降级、限流
    • 多数据中心/多机房容灾策略
    • Chaos Engineering 在 RPC 场景下的应用
  5. 性能调优要点
    • 网络层与序列化层的性能优化
    • 线程池与连接池的优化
    • JIT、JVM 调优与系统级调优
  6. 安全与合规
    • 数据脱敏与加密
    • 授权与审计机制
    • 合规要求（GDPR 等）
