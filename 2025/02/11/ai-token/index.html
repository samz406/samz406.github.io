简单Token及其在GPT模型中的应用

发表于 2025-02-11 分类于 AI

百万token 1元，是怎计算出来的。

模型(1) 上下文长度 最大思维链长度(2) 最大输出长度(3) 百万tokens 输入价格 （缓存命中）(4) 百万tokens 输入价格 （缓存未命中） 百万tokens 输出价格 输出价格
deepseek-chat 64K - 8K 0.5元 2元 8元
deepseek-reasoner 64K 32K 8K 1元 4元 16元(5)

这是截取deep seek的官网API价格的一个描述。这里的token是什么意思呢，下面是官网对token的解释。

token 是模型用来表示自然语言文本的基本单位，也是我们的计费单元，可以直观的理解为“字”或“词”；通常 1 个中文词语、1 个英文单词、1 个数字或 1 个符号计为 1 个 token。

一般情况下模型中 token 和字数的换算比例大致如下：

  • 1 个英文字符 ≈ 0.3 个 token。
  • 1 个中文字符 ≈ 0.6 个 token。

但因为不同模型的分词不同，所以换算比例也存在差异，每一次实际处理 token 数量以模型返回为准，您可以从返回结果的 usage 中查看。

介绍

先介绍一个网站TickTokenizer，https://tiktokenizer.vercel.app/ 。比如我输入“色少时诵诗书少时诵诗书是撒是撒”。输出如下内容。

右下角的Token ID 列表：

  • 200264, 17360, 200266, 3575, 553, 261, 10297, 29186, 200265, 200264, 1428, 200266, 4472, 11389, 1728, 2263, 113, 54710, 20121, 11389, 1728, 2263, 113, 54710, 20121, 3221, 171156, 3221, 171156, 200265, 200264, 173781, 200266

右上角Token ID 代表的文本

  • 这些 token ID 对应着左侧输入的对话：
    • system 角色的 "You are a helpful assistant"
    • user 角色的 "色少时谙诗书少时谙诗书是撒是撒"
  • 特殊标记 <|im_start|>, <|im_sep|>, <|im_end|> 也被编码成了 token ID。

这些是 OpenAI 使用的 tokenizer 将文本转换为整数 ID 后的结果，每个整数 ID 代表一个 token（一个单词、一个子词或一个字符）。这个 token ID 序列会被 GPT 模型 用作输入，神经网络根据这些 token 进行推理、生成下一个 token 并输出文本。

token ID 值是怎么计算

BPE 分词（Byte Pair Encoding）

BPE 是一种基于统计的子词分割算法，用于将文本拆分成常见的 单词、子词或字符片段，然后将它们映射到一个唯一的 token ID。计算步骤如下：

  1. 构建初始词汇表

    • 初始词汇表包含：
      • 单字符
      • 常见单词
      • 特殊标记（如 <|im_start|>，<|im_end|>）

  2. 逐步合并频率最高的字符对

    • BPE 会扫描文本，找到最常见的字符对（例如 t 和 h 组合成 th）。
    • 这个新组合被加入到词汇表，赋予一个新的 token ID。
    • 继续这个过程，直到构建出完整的词汇表。

  3. 编码文本

    • 在编码时，BPE 使用已学习的词汇表来匹配输入文本，并将其转换为 token ID 序列。

    具体计算 Token ID

    Tiktoken 是 OpenAI 采用的分词库，基于 BPE，并针对 GPT 进行了优化。计算 Token ID 的流程如下：

    分词计算

    比如如下文本：

    You are a helpful assistant

    BPE 分割：

    [“You”, “ are”, “ a”, “ helpful”, “ assistant”]

    Token ID 映射

    最终，这句话的 Token ID 表示如下：

    汉字分词

    汉字通常是一个汉字对应一个 token，但如果是常见词组，可能会被编码为一个 token。

    示例：

    enc.encode(“色少时谙诗书少时谙诗书是撒是撒”)

    可能会得到：

    [54710, 20121, 11389, 1728, 2263, 113, 54710, 20121, 3221, 171156, 3221, 171156]

    这里：

    • "色少" → 54710
    • "时谙" → 20121
    • "诗书" → 11389
    • "是撒" → 171156
不是所有单词都有单独的 Token ID

GPT 使用的是 BPE（Byte Pair Encoding）分词算法，它的词汇表包含：

  • 常见单词（独立 token）

    ，如：

    • "Hello" → 9906
    • "world" → 2781

  • 常见汉字（独立 token）

    ，如：

    • "你" → 1762
    • "好" → 1929

  • 长单词会被拆分

    （特别是罕见单词），如：

    • "helpfulness" → [10297, 283]（helpful + ness）
    • "unbelievable" → [1127, 289, 1812]（un + believ + able）
特殊字符和空格也有 Token ID

GPT 使用的分词器会把 空格、标点、特殊标记 作为独立 token：

可能得到：

  • "Hello" → 9906
  • "world" → 2781
  • "!" → 0（标点是独立 token）

空格也是 Token

可能得到：

[259, 9906] # 259 是空格

所以 " Hello" 和 "Hello" 不是同一个 token。

GPT 如何预测下一个 Token

GPT 是一个 自回归 Transformer 模型，它按照以下步骤预测下一个 Token：

(1) 输入 Token ID 序列

例如：

[200264, 17360, 200266, 3575, 553, 261, 10297]

这个 Token 序列可能对应：

(2) 传入 Transformer 计算

  • 这些 Token ID 会被转换为词向量（Embedding）。
  • Transformer 层 计算当前上下文的语义信息。
  • 注意力机制（Self-Attention）让模型聚焦于重要的上下文单词。

(3) 输出概率分布（Softmax

  • GPT 不会直接输出下一个 Token，而是会计算 每个可能 Token 的概率。

  • 例如：

  • GPT 选择概率最高的 Token ID (10297) 作为下一个 Token。

预测 Token ID 的具体流程

如果我们有输入：

“You are a”

步骤 计算内容
输入 "You are a"
Token ID 编码 [3575, 553, 261]
Transformer 计算 处理这些 Token，计算下一个 Token 的概率分布
Softmax 预测 预测 helpful (10297) 的概率最高
输出 Token ID [10297]（对应 "helpful"）
追加 Token 并继续 "You are a helpful"，继续预测 "assistant"

这个过程会一直进行，直到模型预测出 结束标记 或达到最大 Token 数。

Token 不是固定的

GPT 预测下一个 Token 不是固定的，因为它是基于概率的模型（同样的输入可能有不同的输出），是通过temperature参数控制的。

  • temperature 参数会影响输出随机性
    • 低温度 (temperature=0.2)→ 选择最确定的 Token，输出更稳定。
    • 高温度 (temperature=1.2) → 选择更加随机的 Token，创造性更强。

所以你要与众不同，改下temperature值试试。
