常见系统指标缩写

指标缩写

缩写 全称 含义
RT Response Time 响应时间（ResquestTime + ResponseTime）
HPS Hit Per Second 每秒（请求）点击数
TPS Transaction Per Second 每秒事务数
QPS Queries Per Second 每秒查询数
RPS Requests Per Second 每秒请求数
PV Page View 页面浏览量
UV Unique Vistor 独立访问者
Throughput 吞吐量
IOPS Input/Output Operations Per Seconde 磁盘的每秒I/O操作数

主流系统大多倾向于使用 HPS 作为首选的限流指标，能够在一定程度上反映系统当前以及接下来一段时间的压力-周志明

系统可用性公式

时间维度：Availability = Uptime / (Uptime + Downtime)

请求维度：Availability = Successful request / Total request

SLA（Service-Level Agreement）

服务等级协议，指的是系统服务提供者对客户的一个服务承诺。衡量一个系统是否“健康”的方法;有4个指标说明

可用性

​ 系统服务能正常运行所占的时间百分比

准确性

​ 准确性指系统服务中，是否允许某些数据是不准确的或者是丢失。如果允许这样的情况发生，用户可以接受的概率，通常以错误率定义。

​ 例如，我们在一分钟内发送 100 个有效请求到系统中，其中有 5 个请求导致系统返回内部错误，那我们可以说这一分钟系统的错误率是 5 / 100 = 5%。

系统容量

​ 在数据处理中，系统容量通常指的是系统能够支持的预期负载量是多少，一般会以每秒的请求数为单位（QPS）来表示。

延迟

延迟是系统在收到用户的请求到响应这个请求之间的时间间隔，在定义延迟的SLA会有p95 、p99 这样的声明。p指的是 percentile百分位。如果一个系统的p95 延迟是1秒的话，那就表示在100个请求里面有95个请求的响应时间会少于1秒，而剩下的 5 个请求响应时间会大于1秒。
