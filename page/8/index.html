MCP介绍

发表于 2025-03-12 分类于 AI

MCP（Message Context Protocol）是一个开放的协议，它标准化了应用程序如何向大型语言模型（LLMs）提供上下文信息。可以将MCP想象成AI应用的USB-C接口。正如USB-C提供了一个标准化的方式来连接你的设备到各种外设和配件，MCP提供了一个标准化的方式来连接AI模型到不同的数据源和工具。

简而言之，MCP通过标准化的方式，使得AI应用能够更容易地与不同的数据源和工具进行交互，类似于USB-C接口使得各种设备能够方便地连接和交换数据。这样的标准化有助于提高AI应用的兼容性和灵活性。

整洁架构之道之组件

发表于 2025-03-11 分类于 架构

在《整洁架构之道》的第四部分，Robert C. Martin（Uncle Bob）探讨了组件的设计原则。这部分内容不仅仅是关于如何组织代码，而是深入到软件架构的本质：如何合理划分系统边界，使其具有可维护性、可扩展性和可演化性。

许多开发者在初学编程时，会专注于类的设计，关注 SOLID 原则，认为只要类设计得好，代码就能保持清晰和可维护。然而，在实际的大规模软件开发中，类的设计往往并不是主要瓶颈，真正决定系统质量的是组件的组织方式。如果组件的划分和依赖关系不合理，即便类设计得再好，也无法拯救一个混乱的架构。

JVM之Volatile

发表于 2025-03-06 分类于 JVM

volatile是Java提供的一种轻量级的同步机制。它不像synchronized那样提供互斥性，但它能确保变量的可见性和一定程度的有序性。简单来说，当一个变量被声明为volatile时，就告诉JVM这个变量可能会被多个线程同时访问，需要特殊处理。

JVM之逃逸分析

发表于 2025-03-06 分类于 JVM

逃逸分析是一种静态编译时分析技术，用于确定对象的作用域是否仅限于方法内部。简单来说，它帮助JVM确定一个对象是否可能“逃逸”出方法的局部作用域，或被其他线程访问。如果JVM判定对象不会逃逸出局部方法，则该对象便可进行特定的优化，例如栈上分配、标量替换以及消除不必要的同步操作。
